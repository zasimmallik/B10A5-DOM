What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the page as a tree of nodes and objects that programs can modify. Through the DOM, JavaScript can access and change a document's structure, style, and content. It provides methods and properties to navigate, add, modify, or delete elements on a webpage.

How do you select an element from the DOM?

There are several methods to select DOM elements:

  • getElementById: Gets element by ID
  • getElementsByClassName: Gets elements by class name
  • getElementsByTagName: Gets elements by tag name
  • querySelector: Gets first element matching CSS selector
  • querySelectorAll: Gets all elements matching CSS selector

Modern approaches favor querySelector and querySelectorAll for their flexibility with CSS selectors.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique where you attach a single event listener to a parent element instead of multiple listeners on child elements. The listener then detects events that bubble up from the children.

Benefits:

  • Improves memory efficiency by using fewer event listeners
  • Works with dynamically added elements without adding new listeners
  • Requires less code and maintenance
  • Enhances performance, especially with many elements

How do you manipulate an element's attributes and styles using the DOM?

Manipulating Attributes:

  • getAttribute(): Gets attribute value
  • setAttribute(): Sets attribute value
  • removeAttribute(): Removes attribute
  • Direct property access: element.id, element.src, etc.

Manipulating Styles:

  • element.style: Sets inline styles directly
  • className: Sets the entire class attribute
  • classList: Adds, removes, or toggles classes
  • getComputedStyle(): Gets computed styles

Best practice: Use CSS classes with classList instead of inline styles to maintain separation of concerns.
